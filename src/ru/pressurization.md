# Pressurization

Герметизация в Archean позволяет создавать закрытые отсеки, которые могут удерживать газы при давлении, отличном от окружающей среды. Это необходимо для строительства кабин космических кораблей, подводных лодок, дирижаблей или пользовательских топливных баков.

## Как это работает

### Компонент Volume

Компонент [Volume](components/fluids/Volume.md) -- это основа системы герметизации. При размещении внутри конструкции и сканировании он использует **воксельный алгоритм** для определения границ замкнутого пространства.

1. **Сканирование**: нажмите кнопку `Scan` на компоненте Volume для анализа окружающего пространства
2. **Распространение**: система расширяется во всех направлениях (X, Y, Z), пока не достигнет твёрдых/герметичных блоков
3. **Статус**:
   - **AIRTIGHT** (зелёный): отсек полностью герметичен
   - **LEAK** (красный): есть отверстие наружу

> Каждый компонент Volume обнаруживает только тот отсек, в котором он размещён. Для каждого изолированного отсека нужен отдельный компонент Volume.

### Герметичные компоненты

Не все компоненты герметизируют объём. Только компоненты, отмеченные как **герметичные**, выступают в качестве стенок для герметизации. К ним относятся:

| Категория | Герметичные компоненты |
|----------|---------------------|
| **Шарниры** | Small Hinge, Small Pivot, Linear Track |
| **Мосты** | Data Bridge, Fluid Bridge, Item Bridge, Low/High Voltage Bridge |
| **Соединения** | Fluid Junction, Item Junction |
| **Хранение** | Container, Small Container |
| **Стыковка** | Dockable Door |
| **Прочие** | Passive Radiator, Aileron, Mini Router... |

Обычные строительные блоки (рамки, панели) также герметичны по умолчанию.

## Шарниры и герметизация

Шарниры ([Small Hinge](components/joints/SmallHinge.md), [Small Pivot](components/joints/SmallPivot.md), [Linear Track](components/joints/LinearTrack.md)) имеют особое поведение:

- **В исходном положении** (нейтральная позиция, значение = 0): шарнир герметичен и запечатывает объём
- **При смещении** (любая другая позиция): шарнир нарушает герметичность, вызывая утечку

Это позволяет создавать **функциональные двери и люки**:

1. Постройте дверную раму из блоков
2. Разместите Small Hinge или Small Pivot
3. Постройте дверную панель на подвижной части шарнира
4. Когда шарнир возвращается в позицию 0, отсек герметизируется

> **Важно**: когда шарнир перемещается, компонент Volume автоматически пересканирует пространство для определения нарушения герметичности.

## Dockable Doors

[Dockable Door](components/docking/DockableDoor.md) имеет встроенное управление герметизацией:

- **Закрыта**: действует как герметичная заглушка
- **Открыта**: позволяет обмен газом между:
  - Внутренним отсеком и внешней средой (если не пристыкован)
  - Двумя пристыкованными отсеками (если пристыкован к другой Dockable Door)

Когда два космических корабля стыкуются и открывают свои Dockable Doors, их атмосферы смешиваются и давление выравнивается.

## Fluid Port -- ключевой интерфейс

[Fluid Port](components/fluids/FluidPort.md) -- это **основной компонент** для взаимодействия с герметизированными объёмами. Это единственный способ программно закачивать или извлекать жидкости из герметичного отсека.

### Как это работает

Fluid Port действует как **мост** между системой жидкостей (насосы, баки, трубы) и герметизированным объёмом:

- **Позиция важна**: сопло порта должно **физически находиться внутри** герметизированного объёма для взаимодействия с ним
- **Автоматическое обнаружение**: он автоматически определяет, находится ли внутри просканированного объёма или в окружающей среде
- **Двунаправленный**: может как закачивать жидкости в объём, так и извлекать из него

### Физические ограничения

Fluid Port имеет жёсткие физические ограничения:

| Свойство | Значение |
|----------|-------|
| **Максимальная скорость потока** | 1,0 м³/с |
| **Расчёт потока** | Ограничен `min(запрошенная_масса / плотность, 1.0 м³/с × deltaTime)` |

Это означает:
- Плотные жидкости передают больше массы в секунду, чем лёгкие (газы)
- Невозможно мгновенно заполнить или опустошить объём -- это занимает время
- Несколько Fluid Port могут использоваться параллельно для увеличения общей скорости потока

### Сценарии использования

#### Заполнение пользовательского топливного бака
1. Постройте герметичный отсек с компонентом Volume
2. Разместите Fluid Port соплом **внутри** объёма
3. Подключите Fluid Port к Fluid Pump и источнику топлива (бак, электролизёр и т.д.)
4. Насос закачивает топливо через Fluid Port в ваш пользовательский бак

#### Извлечение из объёма
1. Разместите Fluid Port внутри герметизированного объёма
2. Подключите его к насосу, настроенному на вытягивание
3. Жидкостный состав внутри объёма извлекается пропорционально

#### Система жизнеобеспечения кабины
1. Создайте герметичную кабину с компонентом Volume
2. Разместите Fluid Port внутри для закачки пригодной для дыхания атмосферы
3. Подключите к источнику кислорода и насосу
4. Используйте второй Fluid Port для отвода CO2 или поддержания давления

### Обнаружение среды

Когда Fluid Port **не** находится внутри герметизированного объёма, он взаимодействует с **окружающей средой**:
- На Земле: забирает атмосферный воздух (смесь N2, O2)
- В воде: забирает H2O
- В космосе: нечего забирать (вакуум)

Нажмите `V` на Fluid Port, чтобы увидеть:
- Текущую плотность среды (кг/м³)
- Состав среды (процент по объёму)

## Физика газов

Система герметизации моделирует реалистичное поведение газов:

### Отслеживаемые свойства
- **Давление** (Паскали/кПа)
- **Температура** (Кельвины)
- **Плотность** (кг/м³)
- **Состав** (O2, N2, H2, CH4, H2O и т.д.)
- **Уровень жидкости** (для хранения жидкостей)

### Перенос газов
- Газы перетекают из области высокого давления в область низкого давления
- Скорость переноса зависит от разницы давлений и размера отверстия
- Температура усредняется при смешивании газов

### Утечки
Когда в отсеке есть утечка:
- Газ уходит в окружающую среду (или поступает из неё)
- Перенос продолжается до выравнивания давлений
- В космосе (вакууме) весь газ в конечном итоге улетучится

## Советы по строительству

### Создание герметичной кабины

1. Постройте замкнутое пространство с блоками со всех сторон
2. Разместите внутри компонент Volume
3. Нажмите `Scan` -- если показывает **AIRTIGHT**, всё готово
4. Если показывает **LEAK**, проверьте конструкцию на наличие щелей

### Частые причины утечек

- Отсутствующие блоки в углах или на рёбрах
- Шарниры, не находящиеся в позиции 0
- Открытые Dockable Doors
- Негерметичные компоненты, размещённые в стенах

### Несколько отсеков

Для сложных конструкций с несколькими герметичными зонами:
- Разместите один компонент Volume на каждый отсек
- Каждый будет отслеживать собственное давление и состав
- Соединённые объёмы (с общим пространством) автоматически обмениваются газами

### Использование объёмов в качестве топливных баков

Компонент Volume может хранить любую жидкость, не только пригодный для дыхания воздух:
- Заполняйте жидким O2, H2, CH4 или H2O
- Используйте с [Fluid Pumps](components/fluids/fluidPump/FluidPump.md) для извлечения топлива
- Неправильная форма вашей конструкции становится формой вашего бака
