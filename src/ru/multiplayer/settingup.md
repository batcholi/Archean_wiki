Archean предлагает мультиплеерный режим, позволяющий открыть одиночную игру для других игроков или настроить выделенный сервер.

# Требования к оборудованию
Для обеспечения корректной работы сервера Archean в мультиплеерном режиме рекомендуется следующая минимальная конфигурация:
- **Процессор**: приоритет на высокую мощность на ядро (i7, Ryzen...)
- **Память**: минимум 8 ГБ ОЗУ

Эти требования могут измениться в зависимости от используемой платформы (Windows, Linux и т.д.) и будущих обновлений игры.

# Сеть
Независимо от выбора операционной системы, необходимо убедиться, что порт `8881` открыт для `UDP` и `TCP`.

> **Примечание**: При игре в одиночном режиме без `accept_remote_connections = yes`, если порт 8881 уже занят, игра автоматически использует случайный доступный порт.

# Настройка сервера

## SteamCMD (рекомендуется)
Сервер Archean доступен через SteamCMD. Это рекомендуемый метод для выделенных серверов, так как он позволяет легко обновлять.

```bash
steamcmd +login anonymous +app_update 3557220 +quit
```

После загрузки запустите исполняемый файл сервера, как описано в разделе **Ручная установка** ниже.

## Ручная установка
Для Linux и Windows доступен автономный исполняемый файл для запуска сервера Archean.

Для запуска сервера просто запустите файл `Archean_server[.exe]`, расположенный в папке `Archean-game`. Сервер запустится в консольном режиме и попросит ввести ряд данных для настройки. Эта информация больше не будет запрашиваться после установки значения `server_public_name` в файле `server.ini`.

## Docker
Для настройки выделенного сервера на Linux вам понадобятся `curl` и `Docker` для выполнения скрипта автоматической установки.

Установка Docker может значительно различаться в зависимости от используемого дистрибутива. Мы рекомендуем обратиться к Wiki вашего дистрибутива для успешной установки.

Для установки сервера с помощью официального скрипта выполните следующие шаги:
```bash
# Download the install script
curl -s -o install_archean_server.sh https://archean.space/assets/user_scripts/run_server_with_docker.sh

# Make it executable
chmod +x install_archean_server.sh

# Run this script to Install or Update the server
./install_archean_server.sh
```
> Примечание: Этот метод можно использовать на Windows через WSL (Windows Subsystem for Linux), однако некоторые пользователи сталкивались с возможными проблемами производительности.

## Открытие доступа к одиночной игре
Вы можете открыть доступ к своей одиночной игре, вручную настроив файл `server.ini` со следующими параметрами:
- `server_online = yes`
- `server_public_name = <Любое имя на ваш выбор>`
- `accept_remote_connections = yes`

После этого каждый раз при нажатии `Play` в лаунчере публичный сервер будет отображаться в списке, и другие игроки смогут присоединиться к вам.
> - Не забудьте открыть порт 8881 (или любой другой порт, указанный в `listen_port`).
> - Вам не обязательно устанавливать параметр `server_online` в `yes`, чтобы другие игроки могли присоединиться. Вы также можете пригласить их, сообщив свой IP-адрес и порт сервера, которые они должны будут ввести вручную в лаунчере игры. Достаточно установить `accept_remote_connections = yes`.

# Установка модов на выделенный сервер
На выделенном сервере необходимо вручную загрузить моды с помощью SteamCMD и настроить сервер для их обнаружения.

## 1. Загрузка модов через SteamCMD
Используйте SteamCMD для загрузки модов из Workshop. SteamCMD можно установить в любое место вашей системы.

```bash
# Download a mod by its Workshop ID (replace WORKSHOP_ID with the actual ID)
steamcmd +login anonymous +workshop_download_item 2941660 WORKSHOP_ID +quit
```

где `2941660` -- это Steam AppID игры Archean, а `WORKSHOP_ID` -- идентификатор мода в Workshop (можно найти в URL мода в Workshop).

По умолчанию SteamCMD загружает контент Workshop в:
- **Linux**: `~/.local/share/Steam/steamapps/workshop/content/2941660/`
- **Windows**: `<SteamCMD_install_path>/steamapps/workshop/content/2941660/`

Вы также можете указать пользовательский каталог загрузки с помощью опции `+force_install_dir`.

## 2. Настройка external_mods_dir
В файле `server.ini` установите `external_mods_dir`, указав каталог с загруженными модами:

```ini
[server]
external_mods_dir = /path/to/steamcmd/steamapps/workshop/content/2941660/
```

Сервер просканирует этот каталог на наличие папок модов, содержащих файл `repository`.

## 3. Включение модов для вашего мира
Отредактируйте файл `mods.txt` в папке вашего мира:
```
Archean-data/server/worlds/<world_name>/mods.txt
```

Добавьте по одному имени мода на строку. Имя мода определяется в файле `repository` мода (а не именем папки).

Пример `mods.txt`:
```
MYVENDOR_mymod
OTHERVENDOR_anothermod
```

> - Если мод, указанный в `mods.txt`, не найден, сервер предложит продолжить без него или прервать запуск.
> - Имя мода (формат VENDOR_modname) обычно отображается на странице мода в Steam Workshop.

# Настройка сервера (server.ini)
Файл `server.ini` содержит все настройки сервера. Вот полный список доступных параметров:

## Настройки сервера
| Параметр | По умолчанию | Описание |
|-----------|---------|-------------|
| `server_online` | `no` | Отображается ли сервер в публичном списке |
| `server_public_name` | *(пусто)* | Имя, отображаемое в списке серверов |
| `accept_remote_connections` | `no` | Разрешить удалённые подключения игроков |
| `max_simultaneous_players` | `16` | Максимальное количество одновременных игроков |
| `password` | *(пусто)* | Пароль для приватных серверов (оставьте пустым для публичных) |
| `external_mods_dir` | *(пусто)* | Путь к каталогу с внешними модами |

## Настройки игры
| Параметр | По умолчанию | Описание |
|-----------|---------|-------------|
| `world` | *(пусто)* | Название сохранения мира |
| `spawn` | `earth` | Место появления (небесное тело) |
| `spawn_x`, `spawn_y`, `spawn_z` | Координаты Земли | Координаты точки появления |

## Сетевые настройки
| Параметр | По умолчанию | Описание |
|-----------|---------|-------------|
| `listen_port` | `8881` | Порт для прослушивания |
| `listen_new_connection_timeout_ms` | `500` | Тайм-аут подключения в миллисекундах |
| `automatic_blacklist` | `no` | Автоматически блокировать проблемные подключения |

## Настройки производительности
| Параметр | По умолчанию | Описание |
|-----------|---------|-------------|
| `updates_per_second` | `25` | Частота обновлений сервера (тиков в секунду) |
| `physics_steps_per_update` | `8` | Количество шагов физической симуляции на один тик сервера |
| `auto_save_interval_seconds` | `30` | Интервал автосохранения в секундах |
| `screens_compression_level` | `2` | Уровень сжатия экранов (0-9). Ниже = быстрее CPU, но больше трафика. Выше = медленнее CPU, но меньше трафика |
| `server_loop_thread_cpu_affinity` | `-1` | Привязка потока серверного цикла к ядру CPU (-1 = автоматически) |

> **Совет**: Если на сервере высокая нагрузка CPU от экранов/панелей, попробуйте снизить `screens_compression_level` до 0 или 1 для лучшей производительности за счёт увеличения сетевого трафика.

# Управление администраторами
## Панель администрирования (F4)
Панель администрирования доступна в игре по клавише `F4`.
Она отображает нагрузку сервера и скорость в тиках в секунду. *(25 тиков/с -- максимальная частота по умолчанию, гарантирующая оптимальную производительность)*

Флажок `Admin Privileges` позволяет, помимо прочего, размещать Blueprint бесплатно, когда режим игры установлен на `Adventure`.

Панель имеет две вкладки:
- **Players**: позволяет видеть список подключённых игроков, телепортироваться к ним и/или выгонять их.
- **Entities**: позволяет видеть список сущностей на сервере (с владельцем и алиасом), удалять их и/или телепортироваться к ним. Также показывает общее количество сущностей.

## Добавление/удаление администратора
Для добавления администратора просто измените файл `/Archean-data/server/admins.txt`.
Этот файл должен содержать список идентификаторов игроков, по одному на строку.
Если игрок использует Steam, следует использовать его Steam64 ID.

## Команды администратора
Администраторы имеют доступ к специальным командам чата. Полный список см. в [Chat & Commands](../misc/chat.md), включая команду `/ban`.
