# Getting Started

## Creating a Mod

Моды создаются непосредственно из игры, на вкладке **Mods** в разделе **Local Mods**.

### 1. Reserve a Vendor Name
Нажмите **Reserve Vendor Name**, чтобы выбрать уникальный идентификатор разработчика. Это имя должно содержать **от 3 до 12 буквенно-цифровых символов** и автоматически конвертируется в **ВЕРХНИЙ РЕГИСТР**. Это имя является постоянным и не может быть изменено впоследствии.

### 2. Create a New Mod
После резервирования имени разработчика введите название мода в поле **Create New Mod**. Название мода должно содержать **от 3 до 12 буквенно-цифровых символов** и автоматически конвертируется в **нижний регистр**. Нажмите **Create New Mod** для создания папки мода.

## Mod Folder Structure

После создания мода генерируется следующая структура папок:

```
Archean/Archean-data/mods/
└── MYVENDOR_mymod/
    ├── config.yaml              <- Crafting recipes (optional)
    ├── components/
    │   ├── MyComponent1/
    │   │   ├── MyComponent1 (.bin, .gltf, etc.)  <- 3D model
    │   │   ├── main.xc (optional)                <- XenonCode script
    │   │   └── MyComponent1.png                  <- Thumbnail
    │   └── MyComponent2/
    │       ├── ...
```

Каждый компонент находится в собственной подпапке внутри `components/`. Имя подпапки должно совпадать с именем Entity Root, определённым в Blender.

> Мод может содержать несколько компонентов.

## Creating a Component

Для компонента минимально необходима **3D-модель**, экспортированная из Blender. Скрипт XenonCode и рецепты крафта опциональны.

| Шаг | Описание | Страница |
|------|-------------|------|
| **3D Model** (обязательно) | Создание и экспорт 3D-модели с помощью плагина Archean для Blender | [3D modeling with Blender](blender.md) |
| **XenonCode Script** (опционально) | Добавление поведения с помощью скрипта `main.xc`: экраны, анимации, физика, аудио, порты... | [Functionality with XenonCode](xenoncode.md) |
| **Crafting Recipes** (опционально) | Определение способов крафта, переработки или обработки вашего компонента через `config.yaml` | [Crafting Recipes](config.md) |

## Publishing

Когда ваш мод готов, вы можете опубликовать его в Steam Workshop непосредственно из вкладки **Local Mods**, нажав **Publish** на вашем моде. Префикс разработчика мода должен совпадать с вашим зарезервированным именем разработчика.
