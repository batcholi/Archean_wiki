# 3D modeling with Blender

## Plugin Installation

Вот два способа установки плагина в Blender.

### Method 1: Install from ZIP

1. Перейдите в репозиторий плагина [Archean Blender Plugin](https://github.com/batcholi/archean_blender_plugin)
2. Нажмите зелёную кнопку "Code" и выберите "Download ZIP"
3. Откройте Blender
4. В Blender перейдите в Edit > Preferences > Add-ons
5. Выберите "Install from Disk," затем выберите скачанный ZIP-файл

   <img src="./blender-res/fromdisk.png"/>
6. После завершения установки включите плагин в списке дополнений.

### Method 2: Install by cloning the repository
1. Откройте терминал на вашей системе.
2. Клонируйте репозиторий плагина в папку дополнений Blender, выполнив:
   ```bash
   git clone https://github.com/batcholi/archean_blender_plugin <addons_path>
   ```
3. Запустите Blender и убедитесь, что плагин появился в списке дополнений.
4. При необходимости включите плагин.

<font color="orange">Для пользователей Windows:</font> Установите **Git** и используйте `Git Bash` для клонирования репозитория. В командной строке (CMD) Git не будет распознан, если путь к исполняемому файлу не был добавлен в переменную окружения.

## Plugin Overview

Плагин добавляет два новых элемента в Blender:
1. В меню "Add" в режиме Object появляется новый тип объекта **Archean Entity**, который добавляет базовую структуру для создания нового компонента.

## Using the plugin

Сущность Archean всегда состоит из определённой структуры. Вот её элементы:

*Элементы, отмеченные <font color="green">зелёным</font>, обязательны, отмеченные <font color="orange">оранжевым</font> — опциональны.*
- **<font color="green">Entity Root</font>**: Корневой объект сущности. Он критически важен для экспорта и должен всегда присутствовать.
- **<font color="green">Renderable</font>**: Дочерний объект Entity Root. Это объект, видимый в игре. Их может быть несколько, но мы рекомендуем оптимизировать и сохранять их количество минимальным.
- **<font color="orange">Collider</font>**: Дочерний объект Entity Root, определяющий область столкновения. Collider может содержать от 6 до 8 вершин. В сущности может быть несколько коллайдеров, но мы рекомендуем сохранять их количество минимальным для производительности.
- **<font color="orange">Adapter</font>**: Дочерний объект Entity Root, обычно комбинируемый с **Single Arrow**, определяющий точки подключения для кабелей данных, питания, жидкостей или предметов.
- **<font color="orange">Joint</font>**: Дочерний объект Entity Root, обычно комбинируемый с **Single Arrow**, определяющий точки сочленения для анимации частей сущности через перемещение или вращение. Joint становится родителем любого объекта, включённого в него, включая другие Joint.
- **<font color="orange">Target</font>**: Дочерний объект Entity Root, часто комбинируемый с **Single Arrow**, определяющий позицию и направление, которые могут использоваться для добавления функциональности с помощью XenonCode.

### Parameter overview
В зависимости от того, выбрали ли вы Entity Root или один из его дочерних объектов, список доступных настроек меняется.
#### Entity Root menu buttons
- **Export this Entity and Save**: Экспортирует сущность в папку, где сохранён файл .blend, затем сохраняет файл.
- **Generate Thumbnail**: Генерирует миниатюру сущности, которая используется как её иконка в игре.
#### Entity Root parameters
- **Is Entity Root**: Установите этот флажок, чтобы отметить объект как Entity Root. Это разблокирует функции, специфичные для сущностей.
- **Mass (kg)**: Масса сущности в килограммах.
- **Airtight**: Определяет, будет ли сущность герметичной в строительной системе Archean. Помните, что учитывается объём коллайдера, а не отображаемого объекта. Если при экспорте коллайдер отсутствует, игра автоматически создаёт один, охватывающий сущность.
- **Base Plane is Minus Y**: По умолчанию базовая плоскость сущности совпадает с осью -Z. Установите этот флажок, чтобы выровнять её по оси -Y.
- **Export Vertex UVs**: Установите для экспорта UV-координат. Это особенно важно при использовании экранов, текстур и т.д.

#### Child object menu button
- **Create Default Materials**: Archean использует определённую палитру для сущностей, портов и т.д. Нажмите эту кнопку, чтобы автоматически сгенерировать материалы по умолчанию.
#### Child object parameters
- **Is Renderable**: Отмечает, что этот объект будет отрисовываться в игре. Появляется под-параметр **Export Sharp Edges**, позволяющий экспортировать рёбра, отмеченные как "Sharp" в Blender, чтобы они отображались как каркас в голограммах в игре.
- **Is Joint**: Отмечает объект как сочленение. Появляется список под-параметров для включения ограничений вращения и перемещения.
- **Is Target**: Отмечает объект как цель, используемую для функциональности. Его позиция и направление важны в зависимости от использования.
- **Is Collider**: Отмечает объект как коллайдер. Коллайдеры должны быть простыми и содержать от 6 до 8 вершин. Появляется под-параметр **Is Build Block**, чтобы коллайдер также мог действовать как строительный блок, позволяя сущностям или блокам привязываться к нему, оставаясь выровненными по сетке Archean.
- **Is Adapter**: Отмечает объект как точку подключения для кабелей данных, питания, жидкостей или предметов. Выпадающий под-параметр и кнопка **Create Mesh** позволяют генерировать меш разъёма непосредственно.

> Команда разработчиков Archean обычно использует объекты **Single Arrow** для адаптеров, сочленений и целей, поскольку они представляют собой просто позицию и направление.

## Creating your first entity

Первый важный шаг — правильно ориентироваться в 3D-пространстве. В Archean ось Y — вперёд/назад, ось X — влево/вправо, ось Z — вверх/вниз.

<img src="./blender-res/frontback.png"/>

1. Откройте Blender и создайте новую сцену.
2. Удалите всё, что есть в сцене (по умолчанию куб, камера и свет).
3. В меню "Add" в режиме Object добавьте новую **Archean Entity**.

   Этот начальный объект содержит **Entity Root** и простой куб, отмеченный как **Renderable**. Имя Entity Root — это имя сущности, используемое при экспорте и в игре.

   > Имя **Entity Root** не должно содержать пробелов или специальных символов — только буквенно-цифровые символы.
4. Масштабируйте куб до `0.5 x 0.5 x 0.5`, т.е. `50 x 50 x 50 см`, потому что куб по умолчанию слишком велик. *(Это 2 x 2 x 2 метра в игре.)*
5. Сохраните проект в папку перед продолжением, чтобы вы могли экспортировать сущность позже.
   > - Сохраните проект в папку вашего компонента: `Archean/Archean-data/mods/MYVENDOR_mymod/components/MyComponentName/`
   > - См. [Getting Started](getting-started.md) для информации о создании мода и настройке этой структуры папок.
   > - <font color="orange"> Мод может содержать несколько компонентов, каждый в своей папке.</font>
   > - <font color="red">/!\ Имя папки компонента должно совпадать с именем Entity Root.</font>

### Adding the data port
Добавьте объект **Single Arrow** на одну грань куба, чтобы создать порт данных. Сгенерируйте его меш, назначьте правильный материал и объедините его с основным объектом, чтобы не создавать отдельный Renderable. Поскольку порты часто используют сглаженное затенение, примените модификатор "Edge Split" к основному объекту, чтобы избежать визуальных артефактов.

### Adding a joint to rotate Suzanne
Добавьте объект **Single Arrow** на верхнюю часть куба, чтобы определить точку сочленения. Отметьте его как **Is Joint** и включите вращение только вокруг оси Z. Затем сделайте дочерними объектами сочленения — Suzanne (голова обезьяны Blender) и цилиндр для основания.

Поскольку Suzanne может вращаться полностью, установите **-360** как значение **Low**, **0** как **Neutral** (позиция по умолчанию) и **360** как **High**.

### Adding a screen
Для примера создайте объект, который будет служить основой для экрана. Сначала назначьте материал, похожий на дисплей, затем развернитe UV, выбрав грань экрана так, чтобы она заполняла весь UV-редактор.

> Убедитесь, что **Export Vertex UVs** включён на Entity Root, чтобы UV-координаты были экспортированы.
> Внешний вид материала экрана зависит от вас; например, вы можете превратить стеклянную поверхность в экран.

### Adding a collider
В завершение добавьте коллайдер, определяющий область столкновения сущности. Добавьте куб, масштабируйте его, чтобы он охватывал всю сущность, и правильно расположите. Отметьте его как **Is Collider** и скройте в окне просмотра.

### Port Management for XenonCode
Именование портов следует определённому соглашению для удобства их идентификации в XenonCode. Вот как это работает:
- Порты данных должны быть названы **data** в формате `data.index`, или просто `data`, если порт один. Это именование необходимо для доступа через `input.index` и `output.index` в XenonCode.
  Пример с двумя портами данных: назовите их `data.0` и `data.1`. В XenonCode вы обращаетесь к ним через `input.0` и `input.1`.
  Пример с одним портом данных: просто назовите его `data`. В XenonCode вы обращаетесь к нему через `input.0` и `output.0`.

- Все остальные типы портов (Power, Fluid, Item и т.д.) не имеют специальных требований к именованию. Их объявленное имя используется непосредственно в XenonCode.

## Frequently Asked Questions
### Why do I sometimes see a "Fix now" message in the plugin menu?
Объектам необходимо применить масштаб, чтобы избежать проблем с экспортом. Кнопка "Fix now" применяет масштаб ко всем объектам в сцене одновременно. Обычно вы можете предотвратить это сообщение, применяя масштаб объекта с помощью **Ctrl + A** и выбирая **Apply > Scale**.

### The thumbnail orientation doesn't suit me. How do I change it?
Поверните Entity Root, чтобы изменить ракурс миниатюры. Мы рекомендуем никогда не применять это вращение, чтобы можно было легко вернуть сущность к исходной ориентации. Вам нужно пересоздавать миниатюру только при изменении визуального вида.

### Why should I avoid creating too many Renderables?
Движок рендеринга Archean на 100% основан на трассировке лучей, поэтому количество вершин мало значит. Количество объектов, однако, напрямую влияет на производительность. Помните, что `Количество Renderables = renderables на сущность * количество сущностей` в радиусе примерно 100 км, в зависимости от конечного размера сущности. *(Чем больше сущность, тем больше радиус рендеринга.)*

### Is it better to display text as a texture or as a mesh?
Благодаря трассировке лучей, текст в виде меша часто более производителен и выглядит лучше, потому что обеспечивает более высокое визуальное качество и потребляет мало VRAM по сравнению с текстурами.

### I'm used to making low-poly assets for games. Should I do the same for Archean?
Совсем нет — вы можете создавать очень детальные модели. Трассировка лучей обеспечивает исключительно высокое визуальное качество. Забавный факт: Blender, вероятно, вылетит раньше, чем Archean. Развлекайтесь с вершинами!

### What colors do official components use?
При генерации материалов с помощью **Create Default Materials**, **Color1** — это белый цвет, используемый на большинстве компонентов, **Color2** — это металлический серый, а **Body** — чёрный. Остальные материалы имеют говорящие за себя названия.
