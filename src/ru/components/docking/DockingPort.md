# Description
Docking Port — это компонент, позволяющий соединять две конструкции вместе. Соединение обеспечивает передачу данных, энергии, жидкостей и **предметов** между соединёнными конструкциями, но также физически связывает их вместе. Они зафиксированы друг относительно друга.

# Usage
Docking Port не требует питания.
Отдельный разъём данных позволяет управлять стыковочным портом, а другие разъёмы предназначены для подключения кабелей, передающих данные, энергию или жидкости к/от другого стыковочного порта.

### Transfer Mode
Docking Port может работать в двух режимах передачи, настраиваемых через меню GetInfo (клавиша `V`):
- **Fluid Mode** (по умолчанию): Передаёт жидкости между пристыкованными портами
- **Item Mode**: Передаёт предметы между пристыкованными портами

> Оба стыковочных порта должны быть настроены на один и тот же режим для работы передачи.

> Docking Port может стыковаться только с другим Docking Port.

### Использование с псевдонимами (aliases)
Использование стандартных псевдонимов для обоих портов одновременно невозможно, поскольку объект отображает только одно поле псевдонима в информационном окне (`V`). Аналогично, [Router](../computers/Router.md) отображает только одно поле псевдонима на компонент.
Для раздельного использования портов данных с псевдонимами необходимо использовать [Data Bridge](../computers/DataBridge.md) или [Data Junction](../computers/DataJunction.md). Это позволяет назначать псевдонимы этим объектам вместо стыковочных портов.
