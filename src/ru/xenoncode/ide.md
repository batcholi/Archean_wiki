# XenonCode IDE
**XenonCode IDE** — это интерфейс, позволяющий создавать/редактировать скрипты и программы для управления компонентами и системами. Он основан на языке программирования **XenonCode**, написанном специально для этой игры.

## Features
- **Text Editor**: Текстовый редактор с подсветкой синтаксиса для написания скриптов/программ.
- **Node Editor**: Альтернативная система программирования на основе визуальных нод для создания скриптов визуальным и интуитивным способом.
- **Console**: Консоль для отображения информации, отладки и сообщений об ошибках.
- **Toolbar**: Панель инструментов для создания/удаления скриптов/программ, перезапуска системы и доступа к документации, доступной на странице [Archean](archean.md).

## Using the IDE on computers
Кнопка `CODE` на компьютерах открывает доступ к IDE. Оттуда вы можете начать писать скрипты и программы для управления компонентами и системами.

### IDE Library
IDE включает функцию Library, которая позволяет локально сохранять и повторно использовать файлы XenonCode на разных компьютерах. Это полезно для обмена общими функциями, утилитарными скриптами или шаблонами между несколькими внутриигровыми компьютерами без необходимости копирования-вставки кода вручную.

### Additional information specific to computers
#### [Computer](../components/computers/Computer.md)
Computer имеет встроенный сенсорный экран и визуальный BIOS, который отображает системную информацию и доступные программы на жёстком диске при запуске. Вы можете выбрать программу для выполнения, наведясь на неё и нажав клавишу `F`.

#### [Mini Computer](../components/computers/MiniComputer.md)
MiniComputer меньше, чем Computer, и не имеет сенсорного экрана или визуального BIOS. Он нативно выполняет первую программу, созданную на жёстком диске.

## Interface
Интерфейс XenonCode IDE работает во многом как обычная IDE, которую используют программисты, за исключением того, что вместо иерархии файлов все файлы на виртуальном HDD этого компьютера всегда открыты как вкладки.
Вкладки можно перемещать, прикреплять и откреплять от IDE, размещать рядом, сверху-снизу и т.д.
Вы можете иметь несколько открытых IDE (то есть от нескольких внутриигровых компьютеров) одновременно в игре, хотя они не могут быть прикреплены/привязаны друг к другу.

## Creating a custom BIOS
Можно настроить BIOS компьютера, создав файл с именем `main.xc`. Этот файл выполняется первым при запуске компьютера. Вы можете использовать его для изменения внешнего вида BIOS, автоматической загрузки программы или непосредственного выполнения любого типа кода.

## Programs vs Files
Program — это файл, который можно загрузить с помощью `load_program()`, и его имя будет заканчиваться на `*.main.xc` (вам не нужно добавлять это в имя, просто напишите префикс).
File — это файл для включения в программу, он не может быть загружен как самостоятельная программа, но может быть пользовательским BIOS, если назван точно `main.xc` без какого-либо префикса.
Programs можно писать с помощью системы Node или чистого Code, тогда как обычные файлы можно писать только на чистом Code.
Programs могут включать (`include()`) файлы или другие программы, при условии отсутствия конфликтов имён в их содержимом и определения только одной точки входа `init`.

## Switching between modes (visual nodes / code)
Когда вы создаёте программу, она начнётся в режиме Visual Nodes с несколькими начальными нодами IO в качестве примера, которые вы можете удалить.
Система Visual Nodes всегда автоматически генерирует код в фоновом режиме при внесении изменений.
Вы можете "Switch To Code" в любое время, однако он не будет сразу редактируемым.
Чтобы сделать его редактируемым, вы можете "Convert to Code", что просто закомментирует первую строку #NODES, определяющую ноды, и вы сможете редактировать код.
После редактирования в коде, если вы хотите переключиться обратно на ноды, ваш код будет перезаписан нодами, которые у вас были до первоначального переключения на код.
Это означает, что обычно при конвертации в код это необратимо, если только вы не хотите вернуть всё как было.

## Running the code
Внутриигровые компьютеры всегда автоматически перезагружаются при сохранении изменения в текущем файле (CTRL+S).
Сохранение без внесения изменений ничего не сделает, но вы можете принудительно перезагрузить с помощью CTRL+SHIFT+S, который также принудительно сохранит ВСЕ открытые файлы.
При перезагрузке после сохранения изменения компьютер немедленно загрузит предыдущую программу, если она была запущена, без прохождения через BIOS.
Если произошла ошибка, будь то синтаксическая ошибка или ошибка времени выполнения (например, деление на ноль или некорректная индексация), компьютер зависнет и очистит экран, а ошибка появится в консоли. В этот момент вам нужно будет исправить проблему и перезагрузить его, сохранив изменённый файл.

## Persistency
Внутриигровые компьютеры работают на стороне сервера и продолжают работать, когда вы выходите из игры, если вы играете на выделенном сервере.
Файлы сохраняются на HDD сервера в виде обычного текста, включая временный файл для несохранённых изменений, который обновляется при каждом изменении, поэтому вы не потеряете свой код.
Даже если вы не сохранили файл, если ваша игра вылетит, будь вы играете соло или удалённо, вы восстановите его при следующем входе.
Если вы предпочитаете использовать IDE внешнюю к игре, как vscode, вы также можете редактировать файлы непосредственно на сервере, если вы хостите его сами или играете соло.
Виртуальные HDD хранятся в `Archean-data/server/worlds/<WORLD_NAME>/ARCHEAN_computer/HDD-*` и содержат файлы .xc, как вы их создали, а также подкаталог `storage`, который будет содержать storage-переменные в отдельных файлах со значениями в виде обычного текста, разделённого новыми строками для массивов.
При редактировании файла извне игра обнаружит любые изменения, внесённые в текущую запущенную программу, и автоматически перезагрузит компьютер.
Редактирование файла storage, однако, не вызовет перезагрузку, и значения не будут использованы немедленно, так как они кэшируются в памяти компьютера при загрузке.
Файлы storage будут немедленно обновлены на HDD сервера при изменении значения.

## Collaboration
IDE является совместной, что означает, что несколько игроков могут как минимум просматривать содержимое, тогда как один игрок за раз может редактировать код.
Когда редактирующий игрок вносит изменение, даже если не сохранённое, другие игроки увидят изменение в реальном времени, будь то в режиме Code или Nodes.
Не-редактирующие игроки имеют доступ только для чтения, они не могут редактировать код или ноды, пока не нажмут Edit, после чего они станут текущим редактором, а предыдущий редактор станет наблюдателем только для чтения.

## Shortcut keys
- `CTRL+S` Сохранить текущий файл и перезагрузить компьютер
- `CTRL+SHIFT+S` Сохранить все файлы и перезагрузить компьютер
- `CTRL+D` Выделить следующее вхождение в коде, или дублировать выбранные ноды в режиме Nodes
- `CTRL+C` Копировать выбранный код или ноды
- `CTRL+V` Вставить код или ноды
- `CTRL+N` Создать новый File
- `CTRL+SHIFT+N` Создать новый Program
- `CTRL+Q` Закрыть IDE
