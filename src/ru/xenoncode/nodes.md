# Visual Nodes Editor
Редактор нод XenonCode IDE предлагает широкий спектр нод для создания скриптов и программ.

Он достаточно полный для выполнения сложных задач и также проще для понимания начинающими. Для ещё более продвинутых возможностей вы также можете использовать [XenonCode](documentation.md) в текстовом режиме (code).

---
## Code generation

Система Visual Nodes всегда автоматически генерирует код в фоновом режиме при внесении изменений.
Код генерируется при добавлении нод `output`, `print` или `execute`, и он будет распространяться в обратном направлении через входные зависимости.
Это означает, что добавление ноды input не сгенерирует никакого кода, но добавление ноды output сгенерирует код для обеих нод, когда вы соедините их вместе.

## List of nodes

Вот список различных типов нод, доступных в редакторе, с их описаниями при необходимости:

`Constant Number` Определяет постоянное числовое значение.

`Constant Text` Определяет постоянное текстовое значение.

`Variable Number` Определяет переменное числовое значение, которое может быть задано и сброшено другой нодой.

`Variable Text` Определяет переменное текстовое значение, которое может быть задано и сброшено другой нодой.

`Comment` Позволяет отображать блок комментариев в вашем скрипте.

---
### Input
`By Alias` Использовать выходную ноду компонента по его псевдониму при связи через Router.

`0,1,2,3...` Использовать выходную ноду компонента, подключённого к порту компьютера. При подключении компонента к порту он автоматически обнаруживается компьютером.

---
### Output
`Print (Console log)` Отображает число/текст в консоли компьютера.

`By Alias` Использовать входную ноду компонента по его псевдониму при связи через Router.

`0,1,2,3...` Использовать входную ноду компонента, подключённого к порту компьютера. При подключении компонента к порту он автоматически обнаруживается компьютером.

---
### Math
Математические ноды могут иметь более двух входов, что позволяет объединять несколько значений в одной операции.

`Add` Возвращает сумму двух или более значений.

`Subtract` Возвращает разность двух или более значений.

`Multiply` Возвращает произведение двух или более значений.

`Divide` Возвращает частное двух или более значений.

`Absolute` Возвращает абсолютное значение числа.

`Negative` Возвращает отрицательное значение числа.

`Modulo` Возвращает остаток от деления двух чисел.

`Floor` Возвращает значение, округлённое вниз до ближайшего целого.

`Ceil` Возвращает значение, округлённое вверх до ближайшего целого.

`Round` Возвращает значение, округлённое до ближайшего целого.

`Fract` Возвращает дробную часть числа.

`Sign` Возвращает знак числа.

`Pow` Возвращает степень двух чисел.

`Log` Возвращает логарифм числа.

`Sqrt` Возвращает квадратный корень числа.

`Clamp` Возвращает значение числа, ограниченное между минимумом и максимумом.

`Step` Возвращает 0, если значение меньше порога, и 1, если больше.

`Smoothstep` Возвращает ограниченную плавную интерполяцию между двумя значениями на основе коэффициента.

`Lerp` Возвращает линейную интерполяцию между двумя значениями на основе коэффициента.

`Sin` Возвращает синус угла (заданного в радианах).

`Cos` Возвращает косинус угла (заданного в радианах).

`Tan` Возвращает тангенс угла (заданного в радианах).

`Asin` Возвращает арксинус числа в радианах.

`Acos` Возвращает арккосинус числа в радианах.

`Atan` Возвращает арктангенс числа в радианах.

---
### Boolean
Булевы значения представлены как числовые значения. 0 означает False, всё остальное означает True.
Булевы ноды могут иметь более двух входов, что позволяет объединять несколько условий в одной операции.

`And` Возвращает 1, если все значения оцениваются как True.

`Or` Возвращает 1, если любое из значений оценивается как True.

`Xor` Возвращает 1, если нечётное количество значений оценивается как True.

`Not` Возвращает противоположное булевому значению.

---
### Compare
`Equal` Сравнивает, равны ли два значения, и возвращает булево значение (0 или 1).

`Greater` Сравнивает, больше ли одно значение другого, и возвращает булево значение (0 или 1).

`Lesser` Сравнивает, меньше ли одно значение другого, и возвращает булево значение (0 или 1).

---
### Functions
`Expression` Позволяет писать более продвинутое математическое выражение, используя функции XenonCode и/или алгебру.

`Switch` Возвращает различные значения в зависимости от входного значения.

`Counter` Возвращает счётчик, увеличивающийся при каждом вызове. Начальное значение, минимум и максимум могут быть определены.

`Pulse` Возвращает 1, когда входное значение изменилось.

`PulseToggle` Возвращает булево значение, переключающееся между True/False каждый раз, когда входное значение изменяется.

`Text` Позволяет писать более продвинутое текстовое выражение.

`PID Controller` Возвращает значение PID-регулирования на основе ошибки, интеграла ошибки и производной ошибки.

`Unit Converter` Конвертирует значение между общими физическими единицами одной категории. Поддерживаемые категории: Angle (Radians, Degrees), Rotation (RPS, RPM), Speed (m/s, km/h, mph, ft/s, knots), Temperature (Kelvin, Celsius, Fahrenheit), Pressure (Pascal, bar, psi, atm), Force (Newton, lbf, kgf).

---
### Execution
`Statement` Позволяет писать чистое выражение XenonCode, такое как вызов функции. Должен выводиться в ноду Conditional или Execute.

`Conditional` Учитывает данное выражение, только если данное условие оценивается как True. Может быть объединён с другими Conditional.

`Execute` Выполняет подключённые выражения в порядке сверху вниз.

`Include` Включает другой скрипт XenonCode в текущий скрипт. Убедитесь, что ваши ноды переменных и констант имеют уникальные имена.

---
### Timing

`Time` Возвращает текущий Unix Timestamp в единицах секунд с точностью до микросекунд.

`Delay` Возвращает булево значение 1 после заданной задержки в секундах после того, как вход Set получил изменение с 1 на 0.

`Blinker` Возвращает чередующееся булево значение между 0 и 1, изменяющееся с заданным временным интервалом в секундах.

`Smooth` Возвращает значение, интерполирующее от предыдущей цели (или 0) к текущему целевому значению с заданной длительностью в секундах после того, как вход Start получил изменение с 1 на 0.
