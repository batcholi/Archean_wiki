# XenonCode - Documentation

**XenonCode** — это легковесный язык программирования, разработанный как высокоуровневый скриптинг внутри игр для виртуальных вычислительных устройств (например, программируемый компьютер внутри игры).

# Capabilities

- Типизированные переменные
- Константы
- Динамические массивы
- Стандартные арифметические операции над числовыми значениями
- Удобная конкатенация и форматирование строк
- Пользовательские функции
- Функции и объекты, определяемые устройством
- Trailing/member-функции
- Встроенные стандартные математические функции
- Встроенные операции ввода/вывода между виртуальными устройствами
- Встроенные объекты типа ключ-значение
- Синхронизированные интервальные функции (таймеры)
- Условные конструкции `if`/`elseif`/`else`
- Циклы `while`
- Циклы `foreach` для массивов
- Циклы `repeat n`
- Циклы `for` с первым и последним индексом

# Sample code

```xc
include "my_functions.xc"

; Declare global storage
storage var $myStorageVar : number
storage array $myStorageArray : number

; Declare global variables
var $myVar = 0
var $myVar2 = "Hello World"
var $myVar3 : number
var $myVar4 : text
array $myArray : number
array $myArray2 : text

; Declare a user-defined function that changes the values of $myVar and $myVar2
function @myFunction($arg1 : number, $arg2 : text)
	$myVar = $arg1
	$myVar2 = $arg2

; Declare a user-defined function that returns a value
function @myFunction2($arg1 : number, $arg2 : text) : number
	return $arg1 + size($arg2)

init
	; Call a user-defined function
	@myFunction(1, "Hello")

	; Call a trailing function
	$myVar.@myFunction2("Hey")

	; Add values to an array
	$myArray.append(1)
	$myArray.append(5)
	$myArray.append(0)

	; Sort an array
	$myArray.sort()

	; Iterate over an array
	foreach $myArray ($index, $item)
		$myVar4 &= $index:text & ": " & $item:text & ", "

	; Output to a virtual device (ie: a console at input port 0)
	output.0 ($myVar4)
	output.0 (text("Sum of values in the array: {}", $myArray.sum))
	output.1 ($myArray.0:text)
	output.1 ($myArray.1:text)
	var $index = 2
	output.1 ($myArray.$index:text)
	
	; Repeat a statement three times
	repeat 3 ($i)
		output.1 ($i)

	; for loops
	for 3,8 ($i)
		; value of $i will go from 3 to 8 inclusively
	
```

## Types of developer

1. `User`: Человек, использующий этот язык для написания скрипта, обычно игрок в игре.
2. `Device`: Реализация, определяющая возможности и доступные функции, обычно конкретный тип программируемого виртуального устройства в конкретной игре.

# Syntax

XenonCode разработан с очень простым синтаксисом и очень точной структурой.

- Каждое выражение должно быть коротким и легко читаемым
- Очень мало необходимых специальных символов
- Менее загадочный, чем большинство других языков
- Отступы определяют область видимости (ТОЛЬКО табуляции, без двусмысленностей)
- Одна инструкция на строку
- Индексация массивов, как в большинстве других языков программирования, начинается с 0, но использует точечную нотацию `arr.0` вместо типичной скобочной нотации `arr[0]`
- 100% нечувствителен к регистру
- Реализация может определять пользовательские функции, объекты и точки входа "Device"

### Types

XenonCode — типизированный язык, но включает только два общих типа, а также массивы каждого из них и объекты, определяемые реализацией.

Общие типы данных, которые может объявлять пользователь:
* `number`
* `text`

Переменная `number` внутренне всегда является 64-битным числом с плавающей запятой, но может также действовать как булево значение, когда её значение равно 1 или 0 (true или false), хотя любое ненулевое значение оценивается как true.

Переменные `text` содержат обычный текст Unicode, хотя их максимальный размер зависит от реализации.
Текстовый литерал определяется внутри двойных кавычек `" "`.
Для использования символов двойных кавычек внутри текста вы можете дублировать их.
Другого механизма экранирования нет. Обратная косая черта `\` просто является частью строки, и реализация может решить использовать её для escape-последовательностей.
```xc
var $myText = "Say ""Hello"" to this text"
```

Объектные типы предназначены для использования реализацией и являются непрозрачными для пользователя, то есть их структура необязательно определена, однако реализация может сделать доступными некоторые member-функции для этих объектов для пользователя.

Хотя это типизированный язык, указание типа не нужно, когда он может быть автоматически выведен компилятором при инициализации. Тип следует указывать только при отсутствии инициализации.

Все пользовательские слова должны начинаться с символа-префикса:
- `$` для переменных
- `@` для функций

### Comments

Комментарии — это строки, начинающиеся с `;`
Выражение кода также может заканчиваться завершающим комментарием.
ПРИМЕЧАНИЕ: `//` устарел и НЕ должен использоваться.

# Limitations
Этот язык разработан для потенциального выполнения на стороне сервера в контексте многопользовательской игры, поэтому из соображений безопасности и производительности есть ограничения на то, что могут делать пользователи.

- Нет типов указателей или ссылок (кроме объектов, определяемых реализацией, которые должны быть очищены реализацией во время выполнения)
- Количество инструкций за цикл может быть ограничено, при превышении которого может произойти виртуальный сбой для пользователя
- Массивы могут быть ограничены в размере во время выполнения, при превышении которого может произойти виртуальный сбой для пользователя
- Рекурсивный стек (вызов функции рекурсивно) разрешён с определённым синтаксисом и ограничен 16 рекурсиями
- Функции ДОЛЖНЫ быть полностью определены ДО их использования, поэтому порядок определения имеет значение (это обеспечивает предыдущее ограничение)

### Per-Virtual-Computer limitations
Эти ограничения определяются для каждой реализации и могут включать несколько вариантов или быть настраиваемыми пользователем

- ROM (максимальный размер скомпилированной программы в виде количества 32-битных слов)
- RAM (максимальное количество локальных, глобальных и временных переменных плюс все локальные и глобальные массивы, умноженные на их размер)
- STORAGE (максимальное количество storage-переменных плюс все storage-массивы, умноженные на их размер)
- Frequency (максимальная частота для таймерных функций и чтения ввода)
- Ports (максимальное количество входов/выходов)
- IPC (максимальное количество инструкций за цикл, одна строка кода может считаться за несколько инструкций)

### Operation on data
- Все функции, включая таймеры, выполняются атомарно, предотвращая гонку данных
- Аргументы функций всегда передаются копией, функция НЕ МОЖЕТ изменять переменные, помещённые в её список аргументов
- Trailing-функции ИЗМЕНЯЮТ значение ведущей переменной
- Присваивание переменных всегда копирует значение для общих типов
- Объекты, определяемые реализацией, всегда передаются по ссылке
- Объекты, определяемые реализацией, не могут быть скопированы, если реализация не предоставляет эту функциональность через функцию устройства
- Деление на ноль приводит к ошибке времени выполнения. Ответственность за учёт этого лежит на пользователе.

### Basic rules
- Переменные могут быть объявлены с помощью `var` и при необходимости им может быть присвоено начальное значение, иначе используется общее значение по умолчанию (0 для number и "" для text)
- Объектные переменные должны быть присвоены при объявлении с помощью конструктора или функции устройства, возвращающей объект этого типа
- Начало выражения с имени переменной (начинающегося с `$`) означает, что мы изменяем её значение
- Если следующее слово после присваивания переменной — `=`, то результат следующего выражения будет присвоен как её значение
- Если следующее слово после присваивания переменной — точка и функция, этот вызов функции считается trailing-функцией
- Вызов trailing-функции ИЗМЕНЯЕТ значение ведущей переменной, которая также используется как первый аргумент функции, который должен быть опущен из списка аргументов при вызове
- Начало выражения с имени функции (начинающегося с `@` для пользовательских функций) означает, что мы вызываем эту функцию и отбрасываем её возвращаемое значение
- Вызов функции НИКОГДА не изменяет значение любого из её аргументов общих типов, переданных в скобках, общие типы всегда передаются копией
- Всё в скобках, перед чем не стоит имя функции, считается отдельным выражением, внутренние левые выражения вычисляются первыми
- Применяются типичные правила приоритета математических операторов

# Valid usage

XenonCode разработан для компиляции в байт-код, который очень быстро разбирается хост-игрой во время выполнения.

## Each line must begin with one of the following first words (with examples):

### Global scope
- `include` Импортирует содержимое другого файла
- `const` объявляет глобальную константу
- `var` объявляет глобальную переменную
- `array` объявляет глобальный массив
- `storage` объявляет storage-переменную или массив, которые будут сохраняться между циклами питания
- `init` Определяет тело функции init, которая выполнится один раз при включении устройства
- `tick` объявляет тело функции tick, которая выполняется при каждом тактовом цикле
- `function` объявляет пользовательскую функцию
- `recursive function` объявляет пользовательскую функцию, которая может вызывать себя рекурсивно
- `timer` Определяет тело функции, которая будет выполняться повторно с определённой частотой в Гц
- `input` Определяет функцию ввода
- `;` Комментарии
- точка входа, определённая реализацией
- одна или более табуляция, означающая, что мы находимся внутри тела функции, тогда применяются следующие правила:

### Function body / Entry point
- `var` Объявляет новую переменную в этой локальной области видимости (недоступна извне этой функции)
- `array` Объявляет новый массив в этой локальной области видимости (недоступен извне этой функции)
- `$` Для присваивания нового значения существующей переменной
- `@` Для вызова пользовательской функции
- `output` Встроенная функция для отправки данных другому устройству через определённый порт
- `foreach` перебирает все элементы массива
- `repeat` повторяет блок кода n раз
- `while` выполняет цикл, пока условие оценивается как true
- `for` выполняет блок кода с заданным первым и последним индексом
- `break` останавливает цикл, как если бы он завершил все итерации
- `continue` останавливает текущую итерацию цикла и немедленно запускает следующую
- `if` выполняет блок кода, если условие оценивается как true
- `elseif` после if, когда начальное условие оценивается как false и новое условие оценивается как true
- `else` после if, когда все условия оценились как false
- `return` останавливает выполнение этой функции и возвращает значение вызывающему

## Reserved keywords
Поскольку все пользовательские слова должны начинаться с `$` или `@`, нет необходимости в зарезервированных словах.
Реализация/устройство должны следить за тем, чтобы определять только имена функций и типы объектов, которые не конфликтуют со встроенными ключевыми словами для используемой версии XenonCode.

## Declaring a constant
Константы — это именованные значения, которые никогда не должны меняться во время выполнения программы. Это фиксированные значения, определяемые во время компиляции.
Их присвоенные значения должны быть явно указаны и должны быть выражениями, результат которых может быть определён во время компиляции.

`const $stuff = 5` объявляет константу с именем $stuff со значением 5
`const $stuff = "hello"` объявляет константу с именем $stuff с текстовым значением "hello"

## Declaring a variable
Переменные — это именованные значения, которые могут изменяться во время выполнения программы.
Мы можем присвоить или не присвоить им начальное значение.
Если мы не присваиваем начальное значение, используется значение по умолчанию, и мы должны указать тип.
Переменная доступна только в области видимости, в которой она была объявлена. Например, если мы объявляем переменную в начале функции, она доступна во всей функции. Если мы объявляем переменную внутри блока if, она доступна только внутри этого блока, до соответствующего `else`, `elseif` или до возврата в родительскую область видимости.
Переменная, объявленная в глобальной области видимости, доступна отовсюду.
Для переменных, объявленных в глобальной области видимости, при присваивании начального значения данное выражение должно определяться во время компиляции.
Имена переменных должны начинаться с буквы или подчёркивания (`a`-`z` или `_`), затем должны содержать только буквенно-цифровые символы и подчёркивания.

`var $stuff = 5` объявляет числовую переменную с начальным значением 5 при запуске программы
`var $stuff = "hello"` объявляет текстовую переменную с начальным значением "hello" при запуске программы
`var $stuff:number` объявляет числовую переменную с начальным значением 0 при запуске программы
`var $stuff:text` объявляет текстовую переменную с начальным значением "" при запуске программы
`var $stuff = position()` объявляет экземпляр объекта типа `position`, определяемого реализацией (это пример)

Объекты, определяемые реализацией, не могут быть объявлены без инициализации, так как у них нет значения по умолчанию.

## Assigning a new value to a variable
Для присваивания нового значения переменной мы можем просто начать выражение с имени переменной, за которым следует `=` и выражение, результат которого станет новым значением.
Мы также можем использовать trailing-функцию, которая по своей природе изменит значение указанной переменной.

`$stuff = 5` присваивает значение 5 переменной с именем $stuff
`$stuff = $other + 5` присваивает результат выражения ($other + 5) переменной с именем $stuff
`$stuff.round()` вызывает trailing-функцию, которая округляет значение переменной

## Declaring an array
Массив — это динамический список значений определённого типа. Мы можем добавлять или удалять значения, получать доступ к конкретному значению из списка или перебирать все его значения.
При объявлении массива мы не можем указать начальное значение и должны указать тип.
Массивы инициализируются с нулевым размером при запуске программы, значения могут быть добавлены/удалены/изменены во время выполнения программы.

`array $stuff:number` объявляет массив чисел
`array $stuff:text` объявляет массив текстов

Массивы не могут содержать объекты, определяемые реализацией, только общие типы.

## Declaring storage
Storage используется для сохранения некоторых данных между циклами питания и даже через перекомпиляцию.
Мы можем объявить storage-переменные и массивы типа number или text.
Storage СЛЕДУЕТ объявлять только в глобальной области видимости.
```xc
storage var $stuff:number
storage var $stuff:text
storage array $stuff:number
storage array $stuff:text
```

## Accessing/Assigning the nth item within an array
Для доступа или изменения значения конкретного элемента массива мы должны использовать оператор trail `.`, за которым следует 0-индексированный номер элемента или переменная, содержащая этот индекс
`$stuff.0 = 5` Присваивает значение 5 первому элементу массива
`$stuff.$index = 5` Присваивает значение 5 элементу с индексом, определяемым значением $index
`$value = $stuff.1` Присваивает значение второго элемента массива переменной $value

## Accessing/Assigning the nth character within a text variable
Текстовые переменные работают очень похоже на массивы. Мы можем использовать оператор trail `.` для доступа или изменения значения конкретных символов внутри текста.
`$myText.0 = "a"` Устанавливает "a" как первый символ $myText

### Key-Value objects

XenonCode поддерживает собственный тип ключ-значение, который всегда хранится как текст.
Просто объявите текстовую переменную и присваивайте/считывайте её члены, используя ключ как trailing-член
```xc
var $myObject = ".a{5}.b{8}" ; you can use the serialization format like so, but you don't have to, you may simply start with an empty text and assign the members one by one
print($myObject.a) ; will print 5
$myObject.b += 2 ; adds 2 to b which was 8 and will now be 10
```

## The Init function
Тело функции Init будет выполнено первым каждый раз при включении виртуального компьютера.
Функция init не может быть вызвана пользователем. Она может быть только определена, и устройство автоматически вызовет её при виртуальном запуске.
```xc
init
    $stuff = 5
    @func1()
    ;...
```

## Tick function
Функция tick выполняется в начале каждого тактового цикла этого виртуального компьютера.
Функция tick не может быть вызвана пользователем. Она может быть только определена, и устройство автоматически вызовет её для каждого цикла.
```xc
tick
    ; This body is executed once per clock cycle at the virtual computer's frequency
```

## Timer functions
Таймерные функции выполняются с указанным интервалом или частотой, но не чаще одного раза за тактовый цикл.
Мы можем указать `interval` — каждые N секунд, или `frequency` — N раз в секунду.
Таймерные функции не могут быть вызваны пользователем. Они могут быть только определены, и устройство автоматически вызовет их в подходящее время.
```xc
timer frequency 4
    ; stuff here runs 4 times per second
timer interval 2
    ; stuff here runs once every 2 seconds
```
Примечание: Если тактовая частота виртуального компьютера медленнее заданного интервала или частоты, эта таймерная функция не будет корректно работать с указанным интервалом или частотой и может выполняться при каждом тактовом цикле.

## Input
Точки входа Input — это способ доступа к информации, полученной от другого устройства.
Они могут выполняться любое количество раз за тактовый цикл, в зависимости от того, сколько данных было получено с предыдущего тактового цикла. Реализация может решить выполнять её только один раз за цикл, используя только последние полученные данные.
Устройства могут иметь верхний предел буфера приёма, определяющий максимальное количество раз, когда функция ввода может быть вызвана за тактовый цикл.
Если этот предел достигнут, в буфере сохраняются только последние N значений.
Точки входа Input подобны пользовательским функциям, содержащим аргументы, но без возвращаемого значения, и также мы должны указать индекс порта.
Индекс порта должен быть указан после ключевого слова input и оператора trail `.`
Индекс порта также может быть указан через константу (должен быть известен во время компиляции).
Аргументы функции должны быть заключены в скобки, и их типы должны быть указаны.
Точки входа не могут быть вызваны непосредственно пользователем. Они могут быть только определены, затем устройство автоматически вызовет их, если данные были получены, в конце тактового цикла.
```xc
input.0 ($arg1:number, $arg2:text)
    $stuff = $arg1
input.$myPortIndex ($arg1:number, $arg2:text)
    $stuff = $arg1
```

## Output
Функция output — это способ отправки данных другому устройству. Эта функция предназначена для вызова как выражение и не может использоваться в глобальной области видимости, как функции input.
Мы также должны передать индекс порта, как и с функцией input, и он также может быть указан через константу, известную во время компиляции.
Мы должны передать список аргументов в скобках (или пустые скобки).
`output.0 ($stuff, $moreStuff)`

## If Elseif Else
Как в большинстве языков программирования, мы можем использовать условные конструкции.
```xc
if $stuff == 5
    ; then run this
elseif $stuff == 6
    ; then run that instead
else
    ; all previous conditions evaluate to false, then run this instead
```

## Foreach loops
Этот цикл перебирает все элементы массива.
Блок кода под этим оператором цикла будет выполнен для каждого элемента массива, по одному.
```xc
foreach $stuff ($index, $item)
    ; we loop through the array $stuff, and we define $index which contains the 0-based index of this item and $item for the current item's value
    ; note that $item is a copy of its value, so modifying the value of $item will not affect the original array $stuff
    ; if we want to persist the modified $item value into the original array, we can use $index to index the element from the array like so:
    $stuff.$index = $item
    ; CAUTION: $index is a reference used internally for the loop, don't modify its value unless you actually want to affect the loop
```
Вы также можете использовать цикл foreach с объектами ключ-значение
```xc
foreach $obj ($key, $value)
    print($key)
    print($value)
```

## Repeat loops
Этот цикл повторяет выполнение следующего блока заданное количество раз.
```xc
repeat 5 ($i)
    ; this block will be repeated 5 times, and $i is the 0-based index of this iteration (first time will be 0, last will be 4)
    ; CAUTION: $i is a reference used internally for the loop, don't modify its value unless you actually want to affect the loop
```
Количество повторений (выше указано как 5) также может быть указано через переменную или константу, но не через выражение

## For loops
Этот цикл аналогичен repeat, но принимает первый и последний индекс вместо количества повторений
```xc
for 3,8 ($i)
	; $i will start at 3, and end with 8, inclusively, for a total of 6
```

## While loops
Этот цикл выполняет следующий блок, пока данное условие оценивается как true.
```xc
while $stuff < 5
    $stuff++
```

## Break
Это ключевое слово используется для остановки цикла, как если бы он завершил все свои итерации.
```xc
while $stuff < 5
    $stuff++
    if $stuff == 3
        break
```

## Continue
Это ключевое слово используется для остановки текущей итерации цикла и немедленного запуска следующей итерации.
```xc
while $stuff < 5
    $stuff++
    if $stuff == 2
        continue
```

## Math Operators
- `+` сложение
- `-` вычитание
- `*` умножение
- `/` деление
- `%` модуль
- `^` степень

- `--` уменьшает значение переменной
- `!!` инвертирует значение переменной (если значение `0`, устанавливает в `1`, иначе в `0`)

## Assignment operators
- `=` просто присваивает следующее значение непосредственно

Следующие операторы вычисляют соответствующую математическую операцию между ведущей переменной и следующим выражением, затем присваивают результат обратно ведущей переменной.
- `+=` сложение
- `-=` вычитание
- `*=` умножение
- `/=` деление
- `%=` модуль
- `^=` возведение в степень
- `&=` конкатенация текста

## Conditional Operators
- `==` равно
- `<>` или `!=` не равно
- `>` больше чем
- `<` меньше чем
- `<=` меньше или равно
- `>=` больше или равно
- `&&` или `and` логическое И
- `||` или `or` логическое ИЛИ
- `xor` также доступен, эквивалентен (!!a != !!b)

## Other operators
- `.` (оператор trail) обращение к под-элементу массива или текста, вызов trailing-функции на ведущей переменной или обращение к члену объекта
- `:` (оператор приведения типа) приведение к другому типу
- `&` (оператор конкатенации) конкатенация текстов
- `!` (оператор NOT) инвертирует булево значение или выражение (ненулевые числа становятся 0, а 0 становится 1)

## Casting (parse variables as another type)

Для приведения существующей переменной к другому типу просто добавьте двоеточие и тип, вот так:
```xc
$someTextValue = $someNumberValue:text
```
Это работает только с общими типами `number` и `text`, не с массивами или объектами.

## String concatenation

Для конкатенации текстов просто разделите все текстовые значения/переменные оператором конкатенации `&` в одном присваивании (не забудьте привести к text при необходимости).
```xc
$someTextVar = "Hello, " & $someName & "!, My age is " & $age:text & " and I have all my teeth"
```

## Include
Вы можете разделить свой проект на несколько исходных файлов.
Для этого вы можете поместить часть кода в другой файл `.xc` и использовать ключевое слово `include` в родительском файле.
```xc
include "test.xc"
```
Это фактически то же самое, что взять все строки из `test.xc` и вставить их в текущий файл на место `include`.
Это можно делать на нескольких уровнях, только убедитесь, что файл не включает сам себя, прямо или косвенно, в этом случае определения внутри него будут конфликтовать и приведут к ошибке компиляции.

# User-Defined Functions

Функции определяют группу операций, которые мы можем вызвать один или несколько раз во время выполнения программы.

Операции, выполняемые в функции, находятся в её теле.

Функции могут иметь аргументы, которые могут использоваться внутри тела, чтобы операции могли варьироваться в зависимости от значения некоторых переменных.

Аргументы функции определяются после имени функции, в скобках, и могут быть типа `number`, `text` или объекта, определяемого реализацией.

Имена функций имеют те же правила, что и имена переменных.

ПРИМЕЧАНИЕ: Функции ДОЛЖНЫ быть полностью определены до их использования. Это означает, что порядок объявления функций имеет значение, и мы можем вызывать только функцию, которая была объявлена ВЫШЕ вызывающего, и функция НЕ МОЖЕТ вызывать саму себя. Это обеспечивает правило "отсутствия рекурсии стека".

### Declaration

Вот несколько примеров объявления функций

Эта функция принимает один числовой аргумент:
```xc
function @func0 ($var1:number)
```

Эта функция принимает два числовых аргумента:
```xc
function @func1 ($var1:number, $var2:number)
```

Эта функция принимает числовой аргумент и текстовый аргумент:
```xc
function @func2 ($var1:number, $var2:text)
```

Эта функция принимает аргумент объектного типа `position`, определяемого реализацией:
```xc
function @func3 ($var1:position)
```

Эта функция принимает числовой аргумент и текстовый аргумент и возвращает числовое значение:
```xc
function @func2 ($var1:number, $var2:text) : number
```

### Body
Тело функции (операции, выполняемые при её вызове) должно быть на следующих строках после объявления, с отступом в одну табуляцию.
Пустые строки внутри тела допустимы и игнорируются компилятором.
Тела функций могут содержать оператор `return`, за которым опционально следует выражение, которое будет использовано для присваивания ведущей переменной в вызывающем выражении.
При возвращении значения тип возврата должен быть указан в конце аргументов, после закрывающей скобки и двоеточия.

```xc
function @func1 ($var1:number, $var2:number) : number
    return $var1 + $var2
```

### Call

Вызов функции выполнит операции в её теле.
Для вызова функции просто напишите имя функции (начинающееся с `@` для пользовательских функций), затем её аргументы в скобках, разделённые запятыми, вот так:
```xc
@func1(4, 6)
```
Здесь мы передали два числовых аргумента, поэтому этот вызов выполняет тело объявления выше.
Конечно, также можно использовать переменные или даже сложные выражения вместо просто числовых литералов в качестве аргументов функции.

#### NOTE:
Пропуск аргументов допустим.
Их значение изначально принимает пустое значение по умолчанию ("" или 0), затем сохраняет то, что было передано или присвоено им для последующих вызовов этой функции.
Изменение значения аргумента внутри этой функции также будет сохранено для следующего вызова этой функции, если этот аргумент опущен.
Поэтому, если нужна концепция значения аргумента по умолчанию, они могут быть присвоены аргументу в теле этой функции после их использования.
Эту концепцию пропуска аргументов можно также рассматривать как концепцию, аналогичную статическим локальным переменным в C++.

### Return value

Функции могут возвращать значение с помощью ключевого слова `return`.
Это возвращаемое значение может быть присвоено переменной вот так:
```xc
$value = @func1(4, 6)
```

# Recursive functions
Рекурсивная функция — это функция, которая вызывает саму себя.
Для определения рекурсивной функции используйте `recursive function` и `recurse` для вызова самой функции.
```xc
recursive function @recursiveFunc($myVar:number)
    if $myVar < 15
        recurse($myVar++)
```
Этот пример рекурсивной функции будет вызывать себя, пока значение $myVar меньше 15.
Рекурсивные функции ограничены 16 рекурсивными вызовами.

Примечание: Ключевое слово `recurse` доступно только внутри рекурсивных функций и используется для вызова самой функции. Список аргументов должен совпадать с определением функции.

# Trailing functions
Любая функция может быть вызвана как trailing-функция, даже пользовательские функции.
Их работа заключается в том, что под капотом ведущая переменная передаётся как первый аргумент этой функции, а затем ей присваивается возвращаемое значение.
При вызове trailing-функции мы должны опустить первый аргумент, так как он автоматически передаёт ведущую переменную как первый аргумент под капотом.
Если определение функции не имеет аргументов, это всё ещё допустимо, хотя мы просто не заботимся о текущем значении ведущей переменной, но мы присвоим ей новое значение.
Определение функции ДОЛЖНО иметь тип возврата, совпадающий с типом ведущей переменной, если это общий тип.
Trailing-функция может быть вызвана на объектах, определяемых реализацией, в этом случае первый аргумент должен быть этого объектного типа, тип возврата отсутствует, и функция НЕ ДОЛЖНА возвращать значение.
Поскольку мы не можем передавать массивы как аргументы функций, массивы могут использовать только свои специально определённые trailing-функции.
```xc
$myVariable.round()
```
```xc
$myVariable.@func1(6)
```
```xc
$myArray.clear()
```

# Built-in functions

### Math
Эти функции определены в базовом языке и принимают один или несколько аргументов.
Trailing-математические функции используют ведущую переменную как первый аргумент и изменяют эту переменную, присваивая ей возвращаемое значение.
- `floor`(number)
- `ceil`(number)
- `round`(number)
- `sin`(number) в радианах
- `cos`(number) в радианах
- `tan`(number) в радианах
- `asin`(number) в радианах
- `acos`(number) в радианах
- `atan`(number) или (number, number) в радианах
- `abs`(number)
- `fract`(number)
- `log`(number, base)
- `sqrt`(number)
- `sign`(number [, default])
- `pow`(number, exponent)
- `clamp`(number, minimum, maximum)
- `step`(edge1, edge2, number) или (edge, number)
- `smoothstep`(edge1, edge2, number)
- `lerp`(a, b, number)
- `mod`(number, divisor) оператор модуля
- `min`(number, number)
- `max`(number, number)
- `avg`(number, number)
- `add`(number, number)
- `sub`(number, number)
- `mul`(number, number)
- `div`(number, number)

### Text functions
- `text`(inputTextWithFormatting, vars ...) Подробности см. в разделе ниже
- `substring`($myText, start, length) возвращает новую строку
- `replace`($myText, searchText, replaceText [, count]) возвращает новую строку со всеми вхождениями `searchText`, заменёнными на `replaceText`, или до `count` вхождений, если указано
- `size`($myText) возвращает количество символов в $myText
- `last`($myText) возвращает последний символ в $myText
- `lower`($myText) возвращает версию $myText в нижнем регистре
- `upper`($myText) возвращает версию $myText в верхнем регистре

#### Formatting
Функция `text` принимает формат в качестве первого аргумента.
Формат — это по сути текст, который может содержать фигурные скобки, которые будут заменены значениями некоторых переменных или выражений.
Пример:
```xc
$formattedText = text("My name is {} and I am {} years old.", $name, $age)
```
Пустые фигурные скобки выше будут заменены соответствующими переменными из следующих аргументов в том же порядке.
Также можно форматировать числовые переменные определённым образом, указывая псевдо-значения внутри фигурных скобок вот так:
- `{}` автоматически отображает только необходимые цифры на основе значения (например: `3` или `123.456`)
- `{0}` округление до ближайшего целого (например: `3` или `123`)
- `{00}` округление до ближайшего целого, но с отображением минимум двух цифр (например: `03` или `123`)
- `{0e}` отображение округлённого целого значения в научной нотации (например: `3e+00` или `1e+02`)
- `{0e.00}` отображение значения в научной нотации с двумя цифрами после запятой (например: `3.00e+00` или `1.23e+02`)
- `{0.0}` округление до одной цифры после запятой (например: `3.0` или `123.5`)
- `{0000.00}` отображение минимум 4 целых цифр и округление до двух цифр после запятой (например: `0003.00` или `0123.46`)

### Trailing functions for Arrays
Эти функции ДОЛЖНЫ вызываться как trailing-функции, и они ничего не возвращают, вместо этого они изменяют массив
- $myArray.`clear`() Очищает массив
- $myArray.`sort`() Сортирует массив по возрастанию
- $myArray.`sortd`() Сортирует массив по убыванию
- $myArray.`append`(value1, value2, ...) Добавляет одно или несколько значений в конец массива
- $myArray.`pop`() Удаляет последнее значение из массива, уменьшая его размер на один
- $myArray.`insert`(index, value) Вставляет новое значение в массив в определённую позицию, сдвигая все последующие значения на один
- $myArray.`erase`(index) Удаляет элемент из массива по определённому индексу, сдвигая все последующие значения на один
- $myArray.`fill`(qty, value) Изменяет размер и заполняет массив заданным размером и указанным значением для всех элементов (это очищает все ранее присутствовавшие значения в массиве)
- $myArray.`from`(other [, separator]) Устанавливает содержимое массива из другого массива или текста. Separator — для разделения определённой строкой (действует только когда other — текст). Эта функция также работает в обратном направлении при выполнении на тексте с массивом и разделителем.

### Trailing members for Arrays
Используя оператор trail `.`, мы также можем получить определённую информацию о некоторых типах переменных.
- $myArray.`size` возвращает количество элементов в $myArray
- $myArray.`min` возвращает минимальное значение в числовом массиве
- $myArray.`max` возвращает максимальное значение в числовом массиве
- $myArray.`avg` возвращает среднее значение в числовом массиве
- $myArray.`med` возвращает медианное значение в числовом массиве
- $myArray.`sum` возвращает сумму всех значений в числовом массиве
- $myArray.`last` возвращает значение последнего элемента массива, также позволяет изменить это значение через присваивание выражения

### Other useful helpers
- `contains`($myText, "str") возвращает 1, если $myText содержит "str", иначе 0
- `find`($myText, "str") возвращает индекс первого символа первого вхождения "str" в $myText, иначе -1, если не найдено
Они также работают с массивами.
- `isnumeric`($myText) возвращает 1, если $myText является числом, иначе 0

### Ternary operator
Тернарный оператор — это сокращённое условное выражение, возвращающее значение в зависимости от оценки условия.
```xc
$myVar = if($cond, $valTrue, $valFalse)
```

### Device functions
Реализация должна определять специфичные для приложения функции устройства.
Вот примеры базовых функций устройства, которые МОГУТ или НЕ МОГУТ быть определены:
- `delta`() возвращает разницу во времени в секундах с последнего выполнения этой функции `delta`
Функции устройства, не требующие аргументов, могут использоваться без скобок при вызове из выражения.

# Compiler Specifications
Этот раздел предназначен для разработчиков игр, которые хотят использовать это в своей игре.

XenonCode поставляется со своей собственной библиотекой парсера/компилятора/интерпретатора и очень простым инструментом командной строки.

## Editor
Рекомендуется, чтобы редактор кода выполнял следующий разбор текущей строки при каждом нажатии клавиши:
- Заменять ведущие последовательности пробелов ровно одной табуляцией больше, чем у предыдущей непустой строки, если она начинается дальше
- Автодополнение слов при нажатии клавиши пробела, если это слово не является существующим символом
    - Написать минимальные оставшиеся символы (общий знаменатель) символов, начинающихся с написанных символов
        - Если это ведущее слово: автодополнение только словами выражений глобальной области видимости
        - Если предшествуют табуляции: автодополнение только словами выражений тела функции
        - Когда первый символ в текущем слове — `$`, автодополнение пользовательской переменной
        - Когда первый символ в текущем слове — `@`, автодополнение пользовательской функции
        - В остальных случаях автодополнение одним из:
            - Аргументов текущей функции/foreach/repeat
            - Встроенных функций
            - Операторов

## Runtime

При включении виртуального компьютера:
- Выполнить тело функции init()

Один тактовый цикл, выполняемый 'frequency' раз в секунду:
- Выполнить все функции ввода, получившие данные с последнего цикла (один раз каждая за цикл)
- Выполнить пользовательские события / точки входа
- Выполнить функцию tick
- Выполнить все таймерные функции последовательно, если их время наступило
- Ожидание в течение Elapsed-1/Frequency секунд

# Testing XenonCode
Вы можете захотеть протестировать XenonCode или попрактиковать свои навыки кодирования.
Есть онлайн-инструмент fiddle на <a href="https://xenoncode.com/">XenonCode.com</a>

В качестве альтернативы вы можете захотеть запустить его непосредственно на своём компьютере.
Для этого XenonCode имеет CLI с командой `-run` для тестирования скриптов в консоли.
Этот репозиторий поставляется с инструментом CLI, расположенным в `build/xenoncode`
Вот как вы можете загрузить и запустить XenonCode:
```shell
# Clone this github repository
git clone https://github.com/batcholi/XenonCode.git
cd XenonCode
# Compile & Run the XC program located in test/
build/xenoncode -compile test -run test
```
Вы можете редактировать исходные файлы .xc в `test/`, затем попробовать снова запустить последнюю строку для компиляции и запуска.
Каталог `test/storage/` будет создан, он будет содержать данные storage (переменные с префиксом ключевого слова `storage`).
Обратите внимание, что эта команда `-run` предназначена для быстрого тестирования языка и будет запускать только функцию `init`.
Также убедитесь, что ваш редактор настроен на использование табуляций, а не пробелов, для корректного разбора отступов.

Если вы хотите интегрировать XenonCode в свой проект на C++, вы можете включить `XenonCode.hpp`.
Дополнительная документация скоро появится, а пока вы можете использовать `main.cpp` в качестве примера, но его использование ещё может измениться.
