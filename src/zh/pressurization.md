# 增压系统

Archean 中的增压系统允许您创建密封舱室，使其能够维持与外部环境不同的气压。这对于建造航天器座舱、潜艇、飞艇或自定义燃料箱至关重要。

## 工作原理

### Volume 组件

[Volume](components/fluids/Volume.md) 组件是增压系统的核心。当放置在建筑内部并扫描时，它使用**基于体素的算法**来检测封闭空间的边界。

1. **扫描**：点击 Volume 组件上的 `Scan` 按钮来分析周围空间
2. **传播**：系统向所有方向（X、Y、Z）扩展，直到碰到实心/气密方块
3. **状态**：
   - **AIRTIGHT**（绿色）：舱室完全密封
   - **LEAK**（红色）：存在通向外部的开口

> 每个 Volume 组件只检测其所在的舱室。每个独立的舱室都需要单独的 Volume 组件。

### 气密组件

并非所有组件都能密封空间。只有标记为**气密**的组件才能作为增压系统的壁面。这些包括：

| 类别 | 气密组件 |
|----------|---------------------|
| **关节** | Small Hinge, Small Pivot, Linear Track |
| **桥接** | Data Bridge, Fluid Bridge, Item Bridge, Low/High Voltage Bridge |
| **接线盒** | Fluid Junction, Item Junction |
| **存储** | Container, Small Container |
| **对接** | Dockable Door |
| **其他** | Passive Radiator, Aileron, Mini Router... |

常规建筑方块（框架、面板）默认也是气密的。

## 关节与增压

关节（[Small Hinge](components/joints/SmallHinge.md)、[Small Pivot](components/joints/SmallPivot.md)、[Linear Track](components/joints/LinearTrack.md)）具有特殊行为：

- **对齐时**（中位，值 = 0）：关节气密，密封空间
- **移动时**（任何其他位置）：关节破坏密封，导致泄漏

这允许您创建**功能性的门和舱口**：

1. 用方块建造门框
2. 放置一个 Small Hinge 或 Small Pivot
3. 在关节的可动部分上建造门板
4. 当关节回到位置 0 时，舱室恢复密封

> **重要**：当关节移动时，Volume 组件会自动重新扫描以检测密封是否被破坏。

## Dockable Door

[Dockable Door](components/docking/DockableDoor.md) 集成了增压管理：

- **关闭**：作为气密密封
- **打开**：允许气体在以下之间流通：
  - 内部舱室与外部环境（未对接时）
  - 两个对接的舱室（与另一个 Dockable Door 对接时）

当两艘飞船对接并打开各自的 Dockable Door 时，它们的大气会混合并均衡压力。

## Fluid Port - 关键接口

[Fluid Port](components/fluids/FluidPort.md) 是与增压空间交互的**核心组件**。它是以编程方式向密封舱室注入或提取流体的唯一途径。

### 工作原理

Fluid Port 作为流体系统（泵、储罐、管道）与增压空间之间的**桥梁**：

- **位置很重要**：端口喷嘴必须**物理上位于**增压空间内部才能与之交互
- **自动检测**：它会自动检测自己是在已扫描的空间内还是在外部环境中
- **双向**：既可以将流体推入空间，也可以从空间中提取流体

### 物理限制

Fluid Port 有硬编码的物理限制：

| 属性 | 值 |
|----------|-------|
| **最大流量** | 1.0 m³/s |
| **流量计算** | 受限于 `min(requested_mass / density, 1.0 m³/s × deltaTime)` |

这意味着：
- 密度大的流体（液体）每秒传输的质量比轻流体（气体）多
- 无法瞬间填满或排空一个空间——需要时间
- 可以并联使用多个 Fluid Port 来增加总流量

### 使用场景

#### 填充自定义燃料箱
1. 建造一个带有 Volume 组件的密封舱室
2. 将 Fluid Port 的喷嘴放在空间**内部**
3. 将 Fluid Port 连接到 Fluid Pump 和燃料源（储罐、电解器等）
4. 泵通过 Fluid Port 将燃料推入您的自定义油箱

#### 从 Volume 中提取
1. 将 Fluid Port 放在增压空间内部
2. 连接到配置为拉取模式的泵
3. 空间内的流体成分按比例被提取

#### 座舱生命保障
1. 建造一个带有 Volume 组件的密封座舱
2. 在内部放置一个 Fluid Port 用于注入可呼吸大气
3. 连接到氧气源和泵
4. 使用第二个 Fluid Port 排出 CO2 或维持压力

### 环境检测

当 Fluid Port **不在**增压空间内时，它与**外部环境**交互：
- 在 Earth 上：抽取大气（N2、O2 混合物）
- 在水中：抽取 H2O
- 在太空中：无法抽取（真空）

对 Fluid Port 按 `V` 可查看：
- 当前环境密度 (kg/m³)
- 环境成分（按体积百分比）

## 气体物理

增压系统模拟真实的气体行为：

### 追踪的属性
- **压力**（帕斯卡/千帕）
- **温度**（开尔文）
- **密度** (kg/m³)
- **成分**（O2、N2、H2、CH4、H2O 等）
- **液位**（用于存储液体）

### 气体传输
- 气体从高压区流向低压区
- 传输速率取决于压差和开口大小
- 气体混合时温度取平均值

### 泄漏
当舱室存在泄漏时：
- 气体逸出到（或从）环境中进入
- 传输持续进行直到压力均衡
- 在太空（真空）中，所有气体最终都会逸出

## 建造技巧

### 创建密封座舱

1. 用方块在所有面上建造一个封闭空间
2. 在内部放置一个 Volume 组件
3. 点击 `Scan`——如果显示 **AIRTIGHT**，则已完成
4. 如果显示 **LEAK**，检查结构中的缝隙

### 常见泄漏源

- 角落或边缘缺少方块
- 关节未处于位置 0
- 打开的 Dockable Door
- 壁面上放置了非气密组件

### 多舱室

对于具有多个密封区域的复杂建筑：
- 每个舱室放置一个 Volume 组件
- 每个组件各自追踪自己的压力和成分
- 连通的空间（共享空间）会自动在它们之间传输气体

### 使用 Volume 作为燃料箱

Volume 组件可以存储任何流体，不仅限于可呼吸空气：
- 填充液态 O2、H2、CH4 或 H2O
- 配合 [Fluid Pump](components/fluids/fluidPump/FluidPump.md) 提取燃料
- 建筑的不规则形状即是您的油箱形状
