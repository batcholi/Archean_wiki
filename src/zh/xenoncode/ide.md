# XenonCode IDE
**XenonCode IDE** 是一个允许你创建/编辑脚本和程序来控制组件和系统的界面。它基于 **XenonCode** 编程语言，这是专门为本游戏编写的。

## 功能
- **文本编辑器**：带语法高亮的文本编辑器，用于编写脚本/程序。
- **节点编辑器**：基于可视节点的替代编程系统，以可视化和直观的方式创建脚本。
- **控制台**：用于显示信息、调试和错误消息的控制台。
- **工具栏**：用于创建/删除脚本/程序、重启系统以及访问 [Archean](archean.md) 页面上可用文档的工具栏。

## 在计算机上使用 IDE
计算机上的 `CODE` 按钮允许访问 IDE。在此，你可以开始编写脚本和程序来控制组件和系统。

### IDE Library
IDE 包含一个 Library 功能，允许你在不同计算机之间本地保存和重用 XenonCode 文件。这对于在多个游戏内计算机之间共享通用函数、实用脚本或模板非常有用，无需手动复制粘贴代码。

### 计算机特定的附加信息
#### [Computer](../components/computers/Computer.md)
Computer 有一个集成触摸屏和一个 UI BIOS，在启动时显示系统信息和硬盘上的可用程序。你可以通过瞄准程序并按 `F` 键来选择要执行的程序。

#### [Mini Computer](../components/computers/MiniComputer.md)
MiniComputer 比 Computer 更小，没有触摸屏或可视 BIOS。它将原生执行在硬盘上创建的第一个程序。

## 界面
XenonCode IDE 的界面很像程序员使用的普通 IDE，不同之处在于，它没有文件层次结构，该虚拟计算机 HDD 中的所有文件将始终作为标签页打开。
标签页可以移动、嵌入和脱离 IDE，并排排列、上下排列等。
你可以同时打开多个 IDE（即来自多个游戏内计算机），但它们不能相互嵌入/吸附。

## 创建自定义 BIOS
可以通过创建名为 `main.xc` 的文件来自定义计算机的 BIOS。此文件是计算机启动时第一个执行的文件。你可以用它来修改 BIOS 的外观，或自动加载程序，或直接执行任何类型的代码。

## 程序与文件
程序是可以使用 `load_program()` 加载的文件，其名称以 `*.main.xc` 结尾（你不需要在名称中添加这个，只需写前缀即可）。
文件用于在程序中包含，不能作为独立程序加载，但当命名为 `main.xc`（没有任何前缀）时可以作为自定义 BIOS。
程序可以使用节点系统或纯代码编写，而普通文件只能使用纯代码编写。
程序可以 `include()` 文件或其他程序，只要它们的内容没有命名冲突，并且只定义了一个 `init` 入口点。

## 在模式之间切换（可视节点 / 代码）
创建程序时，它将以可视节点模式开始，并带有一些初始 IO 节点作为示例，你可以将其删除。
可视节点系统将始终在后台自动生成代码。
你可以随时"切换到代码"，但不会立即可编辑。
要使其可编辑，你可以"转换为代码"，这将简单地注释掉定义节点的第一行 #NODES，然后你就可以编辑代码了。
编辑代码后，如果你想切换回节点，你的代码将被你最初切换到代码之前的节点覆盖。
这意味着通常转换为代码是永久性的，除非你想将其恢复到原来的状态。

## 运行代码
游戏内的计算机在当前文件保存更改时（CTRL+S）将始终自动重启。
不做任何更改就保存不会有任何效果，但你可以使用 CTRL+SHIFT+S 强制重启，这也会强制保存所有打开的文件。
保存更改后重启时，如果之前有正在运行的程序，计算机将立即加载该程序，而不经过 BIOS。
如果发生错误，无论是语法错误还是运行时错误（如除以零或错误的索引），计算机将冻结并清空屏幕，错误将显示在控制台中。此时，你需要修复问题并通过保存更改的文件来重启。

## 持久性
游戏内的计算机都在服务器端运行，如果你在专用服务器上游玩，即使你退出登录它们也会继续运行。
文件以纯文本形式保存在服务器的 HDD 上，包括一个用于未保存更改的临时文件，该文件在每次更改时都会更新，因此你不会丢失代码。
即使你没有保存文件，如果游戏崩溃，无论你是单人游戏还是远程游戏，当你重新登录时都会恢复。
如果你更喜欢使用游戏外部的 IDE（如 VSCode），你也可以直接在服务器上编辑文件，如果你自己托管或单人游戏。
虚拟 HDD 存储在 `Archean-data/server/worlds/<WORLD_NAME>/ARCHEAN_computer/HDD-*` 中，它们包含你创建的 .xc 文件，以及 `storage` 子目录，其中包含存储变量在各自的单独文件中，值为纯文本，数组用换行符分隔。
在外部编辑文件时，游戏将检测对当前运行程序所做的任何更改，并自动重启计算机。
然而编辑 storage 文件不会导致重启，值也不会立即使用，因为它们在加载时缓存在计算机的内存中。
不过 storage 文件在修改值后会立即在服务器的 HDD 上更新。

## 协作
IDE 是协作式的，这意味着多个玩家至少可以查看内容，而一次只有一个玩家可以编辑代码。
每当编辑者进行更改时，即使未保存，其他玩家也会实时看到更改，无论是在代码模式还是节点模式下。
非编辑者是只读的，他们无法编辑代码或节点，直到点击 Edit，此时他们将成为当前编辑者，前一个编辑者将变为只读查看者。

## 快捷键
- `CTRL+S` 保存当前文件并重启计算机
- `CTRL+SHIFT+S` 保存所有文件并重启计算机
- `CTRL+D` 选择代码中的下一个匹配项，或在节点模式下复制选中的节点
- `CTRL+C` 复制选中的代码或节点
- `CTRL+V` 粘贴代码或节点
- `CTRL+N` 创建新文件
- `CTRL+SHIFT+N` 创建新程序
- `CTRL+Q` 关闭 IDE
